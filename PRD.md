# PRD：语音输入系统小工具（自用 · Cursor 场景）

| 属性 | 内容 |
|------|------|
| 文档版本 | v0.2 |
| 最后更新 | 2026-03-29 |
| 状态 | MVP 范围已定：桌面端 + Web Speech API + 向前台输入框连续注入 |

---

## 1. 背景与问题

产品经理在 **Cursor** 中编写 prompt 时，客户端语音输入不可用或体验差，依赖键盘输入成本高、速度慢。需要一种**打开小工具后、尽量少动手**的方式，让口述内容**持续进入 Cursor 的输入框**（聊天 / Composer 等），而不是依赖「网页复制再粘贴」。

## 2. 目标与成功标准

- **产品目标**：提供**自用**的 **Windows 桌面小工具**（托盘或小型窗口），使用 **Web Speech API**（内嵌 **WebView2 / Chromium** 内核）将**中文为主**的口述转为文字；在用户使用前**已将焦点置于 Cursor 目标输入框且光标在末尾**的前提下，通过**每条已定稿识别结果 → 仅将本段新文本注入前台**（实现上可采用**内部剪贴板 + 模拟 Ctrl+V**，用户无需手动复制），使 Cursor 输入框内文字**连续追加**，全程**不在工具内编辑正文**（润色在 Cursor 内完成）。
- **成功标准（定性）**：开始一轮听写前点选 Cursor 输入框后，连续口述多句中文，**Cursor 框内**出现**顺序正确、内容完整**的累积文本；用户**不需要**打开网页、**不需要**手动的「复制」操作（程序内部可使用剪贴板作为跨进程传输手段）。

## 3. 用户与场景

| 项目 | 说明 |
|------|------|
| 用户 | 单人自用（产品经理本人） |
| 典型流程 | 打开小工具 → 授权麦克风（首次）→ **点击 Cursor 中目标输入框，保证焦点与光标在末尾** → 在小工具内开始听写 → 每句 finalized 后自动注入 Cursor → 说完后停止 / 清空 → 在 Cursor 内编辑并发送 prompt |

## 4. 范围说明

### 4.1 本期包含（MVP）

- **运行形态**：Windows **本地桌面应用**（推荐 **Tauri 2 + WebView2** 或 **Electron**，实现阶段选定），可常驻托盘。
- **识别能力**：**Web Speech API**（`SpeechRecognition`），语言以 **中文（zh-CN）** 为主；依赖 Chromium 系内核，**不推荐**作为非 Chromium 浏览器网页的唯一方案（本工具内嵌 WebView2 即可统一环境）。
- **开始 / 停止听写**：明确控制识别会话的启停；停止时行为见 6.2。
- **连续追加（在 Cursor 侧）**：对每个 **final / isFinal** 的识别片段，在**前台窗口为 Cursor 且插入点位于输入框末尾**的假设下，将**本段新文本**注入当前焦点控件，使框内全文随口述**持续增长**（段间默认加换行，见 6.1）。
- **免手复制**：用户不进行「选中 → 复制」；允许实现使用**仅含本段新文本**的内部剪贴板 + **模拟快捷键粘贴**（如对前台发送 Ctrl+V），频率与防抖见 6.2。
- **状态展示**：小工具内**只读**展示最近日志或当前会话累积预览（可选）、麦克风/识别状态、错误信息；**不提供**可编辑正文区域作为主要产品路径。
- **清空**：一键清空**会话状态**（及可选：清空工具内预览）；清空后注入策略见 6.3。
- **无云端 STT、无 API Key**：MVP **不依赖**第三方语音识别云服务，仓库内**无需**语音类 Secret（除非后续迭代另增能力）。

### 4.2 本期不包含

- 页面内富文本编辑、多用户、账号体系。
- **Cursor 插件**或官方未文档化的 Chat/Composer **内部 API** 直连写入。
- **云端语音识别 API**、自建 STT 代理（可作为 **v0.3+** 可选增强）。
- 保证「**全程零点击**」或「**光标任意位置**均能正确插入」——见第 7 节约束说明。
- macOS / Linux（除非后续单独立项）。

## 5. 功能需求明细

| ID | 需求 | 验收要点 |
|----|------|----------|
| F1 | Web Speech 识别 | 在 WebView2 内能稳定将中文口述转为文本；浏览器不支持或权限拒绝时有明确提示。 |
| F2 | 开始 / 停止 | 用户可明确结束一轮听写；停止时完成最后一段注入或状态一致（见 6.2）。 |
| F3 | 连续注入 Cursor | 多句 finalized 依次说出后，**Cursor 输入框内**全文为按时间顺序的追加结果（在焦点与光标假设成立时）。 |
| F4 | 免手复制 | 用户无需执行复制操作；可接受程序内部剪贴板 + 模拟粘贴。 |
| F5 | 只读展示 | 工具内无必填的正文编辑框；可展示状态与可选预览。 |
| F6 | 清空 | 清空后新会话从空开始注入；行为符合 6.3。 |
| F7 | 配置与安全 | MVP 无语音云密钥；若存热键等配置，本地文件或注册表存储，敏感信息不入仓库。 |

## 6. 关键交互与规则

### 6.1 追加与分段格式（默认）

- 以 **SpeechRecognition** 每次返回的 **final** 结果为一段。
- 每段注入前，默认在段首或段间插入 **一个换行 `\n`**（首段可不加前置换行，实现时二选一并文档化），便于在 Cursor 中分段阅读。

### 6.2 注入机制（默认）

- **推荐实现**：对每个 **final** 文本，将**仅本段**字符串写入系统剪贴板，随后对**当前前台窗口**发送 **Ctrl+V**（或等价粘贴）。
- **防抖**：若 final 触发过密，可使用 **300ms～500ms 防抖**，避免剪贴板与按键风暴。
- **停止听写**：在收到最后一段 final 并完成注入后，将状态置为空闲；若有缓冲未 final 的语义，以浏览器 API 行为为准并在 UI 上提示。

### 6.3 清空

- 用户点击「清空会话」：清空工具内维护的**会话缓冲与预览**；**可选**将剪贴板置空，避免误粘贴旧段（与 v0.1 逻辑一致，实现时选定）。

## 7. 非功能需求与已知约束

| 类别 | 要求 |
|------|------|
| 隐私 | 识别走 **Chromium / Web Speech** 链路（具体隐私策略以浏览器引擎为准）；无自研云端录音存储。 |
| 运行环境 | Windows 10/11；需 **WebView2 运行时**（若选 Tauri/Electron 方案）。 |
| 错误处理 | 麦克风拒绝、识别错误、**前台非 Cursor** 或**注入失败**时须有可见提示（至少在小工具界面）。 |
| 可维护性 | 单人使用；README 说明安装、麦克风权限、**使用前须焦点在 Cursor 输入框**、推荐 Chrome 内核版本。 |

**关键约束（须向用户说明）**：

1. **焦点**：注入发生时，**前台窗口**应为 Cursor，且**插入点**在目标输入框内；否则文本可能进入错误控件或应用。  
2. **光标位置**：采用「**仅粘贴本段**」策略时，假定光标在**末尾**；若用户中途在框内点击改变光标位置，后续片段可能插在中间，属可接受行为或后续优化（读出全文再拼接属高复杂度，本期不做硬性要求）。  
3. **系统听写替代**：若用户需要「完全由系统接管往焦点框里写字」，可并行使用 **Win + H**；与本工具二选一或混用，不在 MVP 内统一。

## 8. 技术假设与选型

### 8.1 已确定

- **形态**：Windows **桌面小工具**（托盘 / 小窗）。
- **识别**：**Web Speech API**（内嵌 **WebView2**）。
- **语言**：**中文（zh-CN）** 为主。
- **与 Cursor 集成**：**不依赖** Cursor 扩展 API；采用 **OS 级剪贴板 + 模拟粘贴** 注入前台焦点控件。

### 8.2 待实现选定（开发前确认）

- **壳技术**：**Tauri 2** vs **Electron**（权衡安装包体积、Rust 学习成本、开发速度）。
- **全局热键**：是否支持「全局开始/停止听写」（需注册热键、不与系统/Cursor 严重冲突）。

### 8.3 后续可选（非 MVP）

- 云端 STT（Azure / 国内云等）以提高准确率或离线不可用时的降级（需密钥与代理方案，另立 PRD 补丁或 v0.3）。

## 9. 验收清单（MVP）

1. 在 **Cursor 输入框聚焦且光标在末尾** 的前提下，连续口述多句中文，**框内**全文顺序正确，段间有约定分隔（默认换行）。  
2. 整个过程中用户**未执行手动复制**。  
3. 点击清空后，新会话注入不从属于上一轮内容（与 6.3 一致）。  
4. 麦克风拒绝、识别不可用、前台非预期应用时，小工具有明确提示。  
5. 仓库无语音类云密钥；README 完整。
