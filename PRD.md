# PRD：语音转写网页小工具（自用）

|------|------|

## 1. 背景与问题

产品经理在 **Cursor** 中编写 prompt 时，客户端语音输入不可用或体验差，依赖键盘输入成本高、速度慢。

## 2. 目标与成功标准

- **产品目标**：提供**自用**的网页小工具，通过**云端语音识别**将口述内容转为文字，并支持**连续追加**与**自动写入剪贴板**，便于在 Cursor 中粘贴后再编辑。
- **成功标准（定性）**：连续口述多句中文后，在 Cursor 中一次粘贴即可得到**顺序正确、内容完整**的累积文本，且无需在工具内做正文编辑。

## 3. 用户与场景

| 项目 | 说明 |
|------|------|
| 用户 | 单人自用（产品经理本人） |
| 典型流程 | 打开页面 → 授权麦克风 → 开始说话 → 识别结果自动累积并同步剪贴板 → 切换到 Cursor 粘贴 → 在 Cursor 内编辑 |

## 4. 范围说明

### 4.1 本期包含（MVP）

- 调用**云端语音识别 API**，**中文（zh）** 为主要识别语言。
- **开始 / 停止** 控制识别会话（含录音采集与上传/流式链路的启停，具体形态依赖所选 API）。
- **连续追加**：新识别的**已定稿**文本追加到当前缓冲区末尾，不覆盖历史内容。
- **自动写入剪贴板**：在缓冲区更新后，将**当前完整累积文本**写入系统剪贴板（可采用防抖，见 6.2）。
- **只读展示**：以只读区域展示当前累积全文（或等价展示），**不提供**可编辑正文区域；最终编辑在 Cursor 完成。
- **清空**：一键清空当前累积缓冲区；清空后的剪贴板行为见 6.3。
- **密钥与配置**：API Key 等敏感信息**禁止硬编码**进仓库；通过环境变量、本地配置或部署侧注入（实现阶段选定一种并文档化）。

### 4.2 本期不包含

- 页面内富文本编辑、多用户、账号体系。
- 与 Cursor 插件或深度集成（仅依赖**剪贴板 + 手动粘贴**）。
- 离线识别、本地模型推理（MVP 以云端 API 为准）。

## 5. 功能需求明细

| ID | 需求 | 验收要点 |
|----|------|----------|
| F1 | 云端 STT | 能稳定将中文语音转为文本；失败时有明确错误提示。 |
| F2 | 开始 / 停止 | 用户可明确结束一轮采集；停止时缓冲区状态符合 6.2。 |
| F3 | 连续追加 | 多句依次说出后，只读区全文为按时间顺序的追加结果。 |
| F4 | 自动复制 | 缓冲区变化后，剪贴板内容与只读区全文一致（在同一次「清空」周期内）。 |
| F5 | 只读展示 | 无正文编辑框；可选展示连接状态、错误信息。 |
| F6 | 清空 | 清空后只读区为空；下一轮流式追加从新内容开始。 |
| F7 | 配置安全 | 仓库内无密钥；`.env` 或同类文件列入 `.gitignore`（若采用本地 env）。 |

## 6. 关键交互与规则

### 6.1 追加与分段格式（默认）

- 每段 **API 返回的 finalized（已定稿）** 文本追加到缓冲区。
- 段与段之间默认插入 **一个换行符 `\n`**，便于在 Cursor 中分段阅读；若所选 API 仅提供整段文件结果，则在每次完整返回后同样追加并加换行。

### 6.2 自动写入剪贴板

- 每次缓冲区因**新的 finalized 文本**而更新后，将**完整缓冲区**写入剪贴板。
- 建议使用 **300ms～500ms 防抖**，避免过高频率触发剪写与权限相关提示。
- **停止采集**时：再执行一次**立即同步**到剪贴板，保证与缓冲区一致。

### 6.3 清空后的剪贴板

- 用户点击「清空」后：**同时将空字符串写入剪贴板**，避免 Cursor 中误粘贴上一轮长文本。

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 隐私 | 音频与识别过程依赖所选云服务商；PRD 接受该前提；不向无关第三方暴露密钥。 |
| 浏览器 | 需支持麦克风采集及剪贴板写入（实现阶段注明推荐浏览器与最低版本）。 |
| 错误处理 | 网络错误、API 4xx/5xx、权限拒绝须可见提示，不得静默失败。 |
| 可维护性 | 单人使用，代码结构清晰即可；依赖与运行方式在 README 中说明。 |

## 8. 技术假设与待决策

### 8.1 已确定

- 前端形态：**网页**（静态页或极轻后端均可，若以代理保护 API Key 则需后端）。
- 识别能力来源：**云端语音识别 API**。
- 语言：**中文为主**。

### 8.2 待选型：云端 STT 供应商

以下任选其一或同类服务，在开发前确认：

- Azure Speech、Google Cloud Speech-to-Text、阿里云智能语音、腾讯云 ASR、OpenAI 兼容的 Whisper 类 HTTP API 等。

**选型参考维度**：

- 中文场景准确率与延迟；
- 计费方式与自用预估用量；
- 是否支持**流式**或**分片上传**（影响「边说边出字」的体验与实现复杂度）；
- 合规与数据驻留（若后续有要求可再评估）。

### 8.3 API Key 暴露策略（二选一，实现时敲定）

- **方案 A**：纯前端直连云 API（仅适合厂商允许且 CORS 可行的情况，多数场景不推荐）。
- **方案 B**：自建极简后端或 Serverless 代理，浏览器只调自有接口，密钥仅存服务端（**推荐默认方向**）。

## 9. 验收清单（MVP）

1. 连续口述多句中文，只读区全文顺序正确，段间有约定分隔（默认换行）。
2. 在不清空的前提下，Cursor 中粘贴内容与只读区全文一致。
3. 点击清空后，只读区为空且剪贴板为空（或粘贴无残留旧文）。
4. API 异常或麦克风拒绝时，界面有明确提示。
5. 仓库中无硬编码密钥，配置方式有文档说明。
