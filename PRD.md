# PRD：Cursor 语音 Prompt 输入小工具（自用）

| 属性 | 内容 |
|------|------|
| 文档版本 | v0.2 |
| 最后更新 | 2026-03-29 |
| 状态 | MVP 已定稿：Windows 桌面壳 + Web Speech API + 向 Cursor 连续注入 |

---

## 1. 背景与问题

产品经理在 **Cursor** 中编写 prompt 时，客户端语音输入不可用或体验差，依赖键盘输入成本高、速度慢。期望**尽量少操作**，理想情况下**无需手动复制**，说话内容能**持续进入 Cursor 的输入框**。

## 2. 目标与成功标准

- **产品目标**：提供**自用**的 **Windows 系统级小工具**（托盘或小型窗口 + 全局热键），通过 **Web Speech API**（运行于 **WebView2 / Chromium 内核**）将**中文为主**的口述转为文字，并在用户将焦点置于 **Cursor 聊天/Composer 输入框**后，把每一句**已定稿（final）**的识别结果**追加写入该输入框**。
- **成功标准（定性）**：
  - 用户**开始一轮说话前**将光标置于 Cursor 输入框末尾（**一次焦点**）；
  - 连续口述多句中文后，**Cursor 输入框内**文本为按时间顺序的追加结果，**用户全程无需在工具与 Cursor 之间手动复制/粘贴**；
  - 最终润色仍在 Cursor 内完成，**工具内不提供**正文编辑区（可选只读状态展示）。

## 3. 用户与场景

| 项目 | 说明 |
|------|------|
| 用户 | 单人自用（产品经理本人） |
| 典型流程 | 启动小工具 →（可选）设全局热键 → **点击 Cursor 输入框**使焦点与光标在末尾 → 在小工具中开始识别/按住说话 → 每句 final 后**由程序自动**将**本句新增文本**注入当前前台输入框 → 在 Cursor 内发送或继续编辑 |

### 3.1 与系统听写的关系（可选替代）

- Windows **Win + H** 语音听写可在焦点控件内持续上屏，**零开发**。若体验已满足，可不必使用本工具；本 PRD 面向需要**自研界面、可扩展（如后续换识别引擎）**的场景。

## 4. 范围说明

### 4.1 本期包含（MVP）

- **形态**：**Windows 桌面应用**（推荐 **Tauri 2 + WebView2** 或 **Electron**，实现阶段选定），可**最小化到托盘**。
- **识别引擎**：**Web Speech API**（`SpeechRecognition`），**中文（zh-CN）** 为主；依赖 Chromium 系内核，**不引入云端 STT 费用**（识别链路以浏览器引擎为准，隐私模型见第 7 节）。
- **开始 / 暂停 / 停止**：用户可明确控制识别会话启停；停止时行为见 6.2。
- **连续追加**：
  - 应用侧维护**逻辑缓冲区**（用于可选只读展示与排错）；
  - 向 **Cursor 注入**时，对每一句 **final** 文本：将**仅本句**（必要时含约定分隔符）写入系统剪贴板，并对**当前前台窗口**发送 **Ctrl+V**（或等价粘贴），从而在**光标处**追加；**不要求用户手动执行复制**。
- **分隔规则**：句与句之间默认追加 **一个换行 `\n`**（可先写入剪贴板为「`\n` + 本句」或「本句 + `\n`」，实现时统一即可）。
- **只读展示（可选）**：可展示当前会话已识别全文或状态（监听中 / 错误）；**不提供**可编辑正文区域。
- **清空**：一键清空应用内逻辑缓冲区；清空后注入策略见 6.3。
- **全局热键（建议）**：开始/停止监听或切换识别，减少鼠标操作（具体键位实现阶段配置）。

### 4.2 本期不包含

- **云端语音识别 API**、后端代理、API Key 管理（MVP 不做；若后续要提升准确率可另起版本）。
- 页面级**纯网页**形态作为唯一交付物（浏览器无法可靠对 Cursor 模拟粘贴；MVP 以**桌面壳**为准）。
- Cursor **插件**或官方未公开的 Chat API 深度集成。
- 本地 **Whisper 等大模型离线识别**（可作为后续迭代）。
- 多用户、账号体系、自动将焦点抢回 Cursor（**不保证**全程零点击，见 5.2）。

## 5. 功能需求明细

| ID | 需求 | 验收要点 |
|----|------|----------|
| F1 | Web Speech 识别 | 在 WebView2/Chrome 内核下可稳定开启中文识别；失败（权限、不支持）有明确提示。 |
| F2 | 开始 / 停止 | 用户可结束一轮；停止时与缓冲区、注入规则一致。 |
| F3 | 连续追加（逻辑 + 注入） | 多句 final 后，Cursor 框内顺序正确；段间默认换行。 |
| F4 | 免手动的「粘贴」 | 每一 final 后由程序完成**写剪贴板（仅增量或约定格式）+ 对前台发送粘贴**；用户无需 Ctrl+C/Ctrl+V。 |
| F5 | 只读展示（可选） | 无正文编辑框；可有状态与错误信息。 |
| F6 | 清空 | 清空逻辑缓冲区；下一轮流式追加从新内容开始；剪贴板策略见 6.3。 |
| F7 | 焦点约定 | 文档与首次启动说明中明确：**注入前须保证 Cursor 输入框为前台焦点且光标在期望位置**（通常末尾）。 |

## 5.2 约束与风险（须在产品说明中告知用户）

- **光标位置**：若用户在说话过程中移动光标或切换焦点，粘贴可能插入错误位置或写入其他应用；**MVP 不解决**跨应用智能定位，依赖用户自觉保持焦点。
- **Electron 输入框兼容性**：以 Cursor 当前版本为准；若未来输入框禁止粘贴，需改版或退回系统听写。
- **Web Speech 准确度与网络**：以 Chromium 实现为准，可能与云端 ASR 有差异。

## 6. 关键交互与规则

### 6.1 追加与分段格式（默认）

- 以 **SpeechRecognition** 的 **final** 结果作为注入单元；**interim** 可不注入 Cursor，仅可选用于 UI 预览（实现阶段定是否展示）。
- 每句 final 注入后，逻辑缓冲区追加同内容 + 换行（与 PRD 一致）。

### 6.2 注入时机与防抖

- 每个 **final** 片段：**设置剪贴板** → **发送粘贴快捷键**至前台。
- 若 final 触发频率过高，可对**剪贴板写入**做 **300ms～500ms 防抖**；**粘贴**须在防抖结束后执行，避免丢句。
- **停止识别**时：可选再执行一次与当前逻辑缓冲区一致的同步（若仅增量粘贴则未必需要整段重写，实现阶段以不破坏光标为准）。

### 6.3 清空后的行为

- 清空**仅影响应用内逻辑缓冲区**与（可选）只读展示。
- **剪贴板**：可将剪贴板置空或保持不变（实现阶段二选一并文档化）；**不向 Cursor 自动注入空串**，除非产品希望「一键清屏」（非 MVP 必选）。

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 平台 | **Windows 10/11**（x64）；注明依赖 **WebView2 运行时**（若用 Tauri/Electron 需说明安装方式）。 |
| 隐私 | 无自研云端；语音数据处理方式以 **Chromium Web Speech** 及微软/谷歌等底层服务说明为准（须在 README 中引用官方说明摘要）。 |
| 权限 | 麦克风权限、剪贴板访问、全局热键（若使用）需在首次使用时清晰请求或说明。 |
| 错误处理 | 权限拒绝、识别错误、前台非 Cursor 导致误粘贴等须有可见提示或日志（至少一种）。 |
| 可维护性 | 单人使用；构建与运行步骤写入 README。 |

## 8. 技术假设与实现选型

### 8.1 已确定（v0.2）

- **宿主**：Windows **桌面应用**（非纯浏览器页）。
- **识别**：**Web Speech API**，语言 **zh-CN** 为主。
- **注入策略**：**系统剪贴板（程序写入）+ 模拟 Ctrl+V** 至前台焦点控件；用户无需手动复制。

### 8.2 待实现时选定

- **Tauri 2** vs **Electron**（权衡体积、Rust 学习成本与开发速度）。
- **全局热键**默认组合（避免与 Cursor 冲突）。
- **interim** 是否显示在只读区。

### 8.3 后续迭代（非 MVP）

- 云端 STT（阿里云/腾讯云等）以提升准确率；
- 本地 Whisper；
- 更稳健的 UI Automation 注入（评估维护成本后再定）。

## 9. 验收清单（MVP）

1. 在 **Cursor 输入框聚焦且光标在末尾** 的前提下，连续说多句中文，框内文本顺序正确，句间有换行（或约定分隔）。
2. 全程**无需用户**在工具与 Cursor 间执行**手动复制/粘贴**。
3. 停止识别、清空后行为符合第 6 节说明。
4. 麦克风拒绝或 Web Speech 不可用时，有明确提示。
5. README 说明：焦点约定、WebView2、隐私摘要、推荐使用方法。

## 10. 里程碑（建议）

| 阶段 | 交付物 |
|------|--------|
| M1 | 工程脚手架（Tauri 或 Electron）+ WebView2 内 SpeechRecognition 跑通 |
| M2 | final 回调 → 写剪贴板 + 向前台发送粘贴；中文与换行验证 |
| M3 | 开始/停止、清空、可选托盘与全局热键 |
| M4 | 错误处理、README、自用打包/启动方式 |
